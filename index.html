    Integration Options
    Option 1
    REST API (preferred)
    Direct OAuth-authenticated calls. Requires UKG to provision API access for CMFR's tenant. Needs Firebase Function proxy due to browser CORS.
    Option 2
    SFTP File Export
    TeleStaff Cloud pushes CSV/XML roster exports to SFTP on a schedule. A Firebase Function ingests and writes to RTDB. No CORS issues.

    OAuth 2.0 Auth Flow
    Step 1 — Request token:
    POST https://{hostname}/ukgpro/oauth/token Content-Type: application/x-www-form-urlencoded appkey: {YOUR_APP_KEY} grant_type=password &username={USERNAME} &password={PASSWORD} &client_id={CLIENT_ID} &client_secret={CLIENT_SECRET}
    Step 2 — Use token in every call:
    GET https://{hostname}/api/v1/roster/today Authorization: Bearer {access_token} Accept: application/json
    ℹ
    Tokens expire in ~3600s. The module auto-refreshes before expiry. Credentials stored in Firebase at config/integrations/ukg/ — Admin-only write.

    Fallback Strategy (3 Levels)
    Level 1
    Cached Firebase data
    If API call fails, serve last successful response from Firebase RTDB with a stale-data timestamp badge
    Level 2
    Mock data mode
    If Firebase also unavailable (offline), fall back to last in-memory mock. User sees "Offline mode" banner
    Level 3
    Manual override
    BC can enter staffing data directly in Hub, bypassing integrations entirely. Never blocked by connectivity
    Recommended Resilience Improvements (v4.2+)
    ImprovementScopePriorityNotes
    Firebase offline persistenceGlobalHighEnable enablePersistence() — all Firebase data survives page reload & offline sessions
    Stale-while-revalidateAPI callsHighShow cached data immediately, refresh in background, swap quietly — zero loading flash
    Firebase Cloud Function proxyUKG / FDHighServer-side credential holder that proxies both APIs — solves CORS, keeps secrets off the browser
    Retry with exponential backoffUKG / FDMedAuto-retry failed calls at 5s, 15s, 60s before tripping circuit breaker to open state
    Service Worker cachingGlobalMedCache static assets + GET responses in SW; enables full offline shell for apparatus/checks pages
    Write queue for offline editsDaily checksLowQueue check submissions when offline, flush to Firebase when reconnected — no lost data
